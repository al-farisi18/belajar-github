# Dokumentasi GitHub

## 1. Pentingnya Penggunaan Command Line
Command Line digunakan untuk menjalankan perintah Git secara langsung seperti push, pull, dan clone repository. 
Dengan command line, proses menjadi lebih cepat, fleksibel, dan efisien dibandingkan menggunakan GUI.

---

## 2. Langkah-Langkah Push Repository
Push digunakan untuk mengirim file dari komputer ke GitHub.

Langkah-langkah:
1. git init
2. git add .
3. git commit -m "first commit"
4. git branch -M main
5. git remote add origin (link repository)
6. git push -u origin main

Fungsi:
Mengupload project ke repository GitHub.

## 3. Langkah-Langkah Clone Repository
Clone digunakan untuk mengambil repository dari GitHub ke komputer.

Langkah-langkah:
1. Buka Git Bash / Terminal
2. Ketik:
   git clone (link repository)

Fungsi:
Mengambil project dari GitHub agar bisa diedit.

## 4. Langkah Pull dan Push Repository
Pull digunakan untuk mengambil update terbaru dari GitHub.
Push digunakan untuk mengirim perubahan ke GitHub.

Langkah-langkah:
1. git pull origin main
2. git add .
3. git commit -m "update"
4. git push origin main

Fungsi:
Agar file tetap sinkron antara lokal dan GitHub.
